# 第1章 学習ログ：SpringとSpring Bootの関係

## ■ 学習概要

第1章では、

- フレームワークとは何か
- Springとは何か
- Spring Bootとは何か

という、

```text
Spring Boot学習全体の入口
```

となる内容を学習した。

この章はまだ実際のコーディングや環境構築には入っておらず、

```text
「Spring系技術が何を目的として作られたのか」
```

を理解することが中心だった。

また、

- IoC（制御の反転）
- DI（依存性の注入）
- Spring Data
- Spring Security
- Spring Boot

など、

今後頻繁に登場する用語や概念についても概要を学習した。

---

# ■ フレームワークについて理解したこと

フレームワークは：

```text
「よく使う処理の土台」
```

を提供する仕組みであり、

- ログイン機能
- 入力チェック
- DBアクセス

などを毎回ゼロから作らなくてよい点が大きな利点だと理解した。

また、

```text
必要な部分だけ差し替えられる
```

という考え方も重要だと感じた。

---

# ■ ライブラリとの違い

ライブラリは：

```text
開発者側が必要時に呼び出すもの
```

であるのに対し、

フレームワークは：

```text
フレームワーク側が処理の流れを管理する
```

という違いがあることを学んだ。

この考え方が：

```text
IoC（制御の反転）
```

につながっている。

---

# ■ Springについて理解したこと

Springは単一の機能ではなく、

- Spring Framework
- Spring Data
- Spring Security
- Spring Batch
- Spring Cloud

など、

```text
複数の機能群の総称
```

であることを理解した。

また、

```text
DI（依存性の注入）
```

がSpringのコア技術であり、

プログラム部品を柔軟に差し替え可能にすることで：

- 保守性
- 拡張性
- テスト容易性

を高めていることを学んだ。

---

# ■ Spring Dataについて理解したこと

Spring Dataでは：

```java
findByLastName
```

のようなメソッド名から、

```text
検索クエリーを自動生成
```

できることを学んだ。

Repository定義だけで：

- Create
- Read
- Update
- Delete

などのCRUD操作を簡単に実装できる点は非常に便利だと感じた。

---

# ■ 自分で調べて理解した点①

教科書では：

```java
findByLastName
```

というメソッド名を書くだけで、

```text
「名字検索処理を自動生成できる」
```

と説明されていた。

しかし実際には：

```text
何でも自動生成できるわけではない
```

ことを調べて理解した。

このメソッドが正しく動作するためには、

```text
対象エンティティに
lastName フィールドが存在している
```

ことが前提となる。

つまりSpring Dataは：

```text
メソッド名
↓
エンティティのフィールド解析
↓
クエリー自動生成
```

を行っていることが理解できた。

---

# ■ Spring Bootについて理解したこと

Spring Bootは：

```text
Spring開発を簡単・高速化するための仕組み
```

であり、

特に：

```text
「設定より規約」
```

という考え方が重要であることを学んだ。

- デフォルト設定の自動化
- Tomcat組み込み
- 初期設定簡略化

などによって、

```text
環境構築の負担を大幅に削減
```

している。

---

# ■ 自分で調べて理解した点②

最初は、

```text
「結局設定を書くなら、
昔のSpringと何が違うのか？」
```

という疑問を感じた。

Spring Bootは：

```text
XML設定が複雑だった問題を解決するため
```

に作られたにも関わらず、

結局：

```text
application.yml
application.properties
```

などを編集する必要があるため、

```text
「面倒さはあまり変わらないのでは？」
```

と感じた。

しかし調べてみると、

Spring Bootは：

```text
設定を完全になくしたのではなく、
「ほとんど」の設定を自動化した
```

という思想であることが理解できた。

特に：

- デフォルト設定が最初から用意されている
- XMLより設定が圧倒的に簡単
- YAML形式は可読性も高い
- Tomcatなども自動組み込み

などによって、

```text
環境構築速度が大幅に向上
```

していることがわかった。

---

# ■ 全体を通して

第1章はまだ概要中心の内容だったため、

```text
技術的難易度はそこまで高くなかった
```

が、

今後のSpring Boot学習全体に関わる：

- IoC
- DI
- Repository
- 自動設定
- 規約ベース開発

などの考え方が多く登場した。

特に：

```text
Spring Bootは
「全部を自動化する魔法」
ではなく、
「共通部分を強力に自動化する仕組み」
```

であることを理解できたのは大きかった。
